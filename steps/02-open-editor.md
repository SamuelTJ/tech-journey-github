---

## No Installation Required

You do not need to install anything. GitHub has a built-in code editor called **github.dev** — it is Visual Studio Code running directly in your browser.

---

## What Is github.dev?

github.dev is the same VS Code editor that many professional developers use, but it runs in a browser tab. You can:
- Browse and open any file in your repo
- Create new files and folders
- Edit files with syntax highlighting
- Commit your changes — all without leaving the browser

---

## Your Task: Open the Editor

There are two ways to open it:

### Option 1: Press the `.` key

1. Make sure you are on your forked repository page on GitHub.
2. Press the **`.` (period)** key on your keyboard.
3. The page will reload as `github.dev/YOUR-USERNAME/techjourney-github` and the VS Code editor opens.

### Option 2: Change the URL manually

In your browser's address bar, change `github.com` to `github.dev` and press Enter:

```
Before: https://github.com/YOUR-USERNAME/techjourney-github
After:  https://github.dev/YOUR-USERNAME/techjourney-github
```

---

## Getting to Know the Editor

Once the editor opens, you will see:

- **Left sidebar** — the file explorer. Click any file to open it.
- **Main area** — the editor where you read and write files.
- **Bottom bar** — shows your current branch name (it says `main` for now).
- **Source Control icon** — the branching icon on the left sidebar (looks like a small node diagram). This is where you commit changes.

Take a moment to click around and open a few files — like this step file you are reading right now.

---

## Navigating Back to GitHub

To go back to the normal GitHub view at any time, change `github.dev` back to `github.com` in the address bar, or click the GitHub logo in the top-left of the editor.

---

> **CLI Alternative (optional, for the curious)**
>
> Developers who work locally clone the repo to their computer and use a terminal to run git commands. That workflow starts with:
> ```bash
> git clone https://github.com/YOUR-USERNAME/techjourney-github.git
> cd techjourney-github
> ```
> For this curriculum, we are skipping that — the browser editor does the same thing without any setup.
